Choose where tools runtime execute based on your deployment needs.

## RunOn Types

| Type      | Runs On               | Scope        | Lifetime   | Use Case                      |
| --------- | --------------------- | ------------ | ---------- | ----------------------------- |
| **AGENT** | Agent's runtime       | Single agent | Session    | Agent-specific context        |
| **EDGE**  | User-selected runtime | Custom       | Persistent | Local resources, edge devices |

## Decision Matrix

**Use AGENT when:**

- Tool needs agent's session context
- Temporary state during conversation
- Agent-specific configuration
- Lifecycle tied to agent execution

**Use EDGE when:**

- Local filesystem access required
- Hardware-specific operations
- Data residency requirements
- Low-latency access to local resources

## Examples

### EDGE: Weather API

```json
{
  "name": "weather_api",
  "type": "api_rest",
  "runOn": "EDGE",
  "config": {
    "endpoint": "https://api.weather.com"
  }
}
```

**Why EDGE:** Stateless API, all agents use same endpoint

### AGENT: Session State

```json
{
  "name": "conversation_memory",
  "type": "coded_function",
  "runOn": "AGENT",
  "config": {
    "retain_context": true
  }
}
```

**Why AGENT:** Maintains state per agent session

### EDGE: Local Filesystem

```json
{
  "name": "filesystem",
  "type": "mcp_server",
  "runOn": "EDGE",
  "config": {
    "command": "npx",
    "args": ["@modelcontextprotocol/server-filesystem", "/data"]
  }
}
```

**Why EDGE:** Accesses local machine's filesystem

## Configuration

### Setting RunOn Type

In tool configuration:

```json
{
  "tool": {
    "name": "my_tool",
    "runOn": "AGENT" | "EDGE"
  }
}
```

### Runtime Assignment

**AGENT:** Auto-assigned to agent's runtime when agent connects

**EDGE:** User selects specific runtime in dashboard

## Best Practices

**Default to EDGE** for stateless tools

**Use AGENT** only when session context required

**Reserve EDGE** for local resources and compliance

**Document** why specific runOn type chosen
